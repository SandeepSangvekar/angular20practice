16)Encapsulation:

*Server-Side Rendering(SSR) :
 - SSR (Server-Side Rendering) means rendering Angular applications on the server instead of only in the browser.
 - The server generates the full HTML of the page and sends it the the client.
 - The browser then displays content immediately and Angular takes over for interactivity.

*Why do we need SSR?
 - Faster First Load - The user will see a page quickly because the HTML is pre-rendered.
 - SEO Friendly - Search engines(Google, Bing, etc.) can crawl real content.
 - Better Social Media Previews - proper meta tags and content are visible.
 - Improved User Experience - No "blank screen" while Angular is loading.

*Advantages of SSR in Angular 20
 - Faster initial page load
 - Better SEO
 - Works well on slow networks or low-end devices
 - Essential for blogs, e-commerce and content heavy apps

*Hydration in Angular?
 Hydration in Angular means:
 - After the server has already rendered the HTML(via SSR), Angular on the browser reuses that HTML into a live Angular app.
 - It does not re-render the DOM, It just "hydrates" the static HTML into a live angular app.

*Route Guards:
 - There are 5 types of guards we use in Angular.
 1) CanActivate - This helps to decide whether the route can be activated or not 1 (route ko activate krne ke liye authentication check krte hai(check login or not).)
 2) CanActivateChild - This helps to decide whether child routes can be activated or not.
 3) CanDeactivate - This helps to decide whether the route can be deactivated or not. Generally, this is used to warm a user if they are navigating from the component. (kisi page ko leave kar karte time puchta hai are u sure to leave this page)
 4) Resolve - This performs route data retrieval before any activation of the route. (resolve mean kisi page pr jane se phle hm check krte hai data available hai ya nahi)
 5) CanLoad - Checks to see if the user can navigate to the module which is lazily loaded.

 - In short:
 1) CanActivate guard (e.g. it checks route access).
 2) CanActivateChild guard (checks child route access).
 3) CanDeactivate guard (prompt for unsaved changes).
 4) Resolve guard (pre-fetching route data).
 5) CanLoad guard (check before loading feature module).

 - to create guard command: ng g guard guard-name
